# Raspberry Pi security camera dashboard

A small [Next.js](https://nextjs.org) app that lists camera recordings stored in [Supabase](https://supabase.com): paginated rows with video playback, live updates (Realtime + polling), and jump controls for large libraries.

## Works with the Pi capture project

This repo is the **web UI** meant to run alongside **[rpi-security-camera](https://github.com/DCabahug1/rpi-security-camera)** (YOLO + OpenCV on the Pi, FFmpeg for web-friendly MP4, upload to Supabase Storage, insert into `recordings`).

| Topic | [rpi-security-camera](https://github.com/DCabahug1/rpi-security-camera) | This dashboard |
| --- | --- | --- |
| **Runs on** | Raspberry Pi (Python) | Anywhere with Node (laptop, Vercel, …) |
| **Supabase key** | **Service role** on a trusted device (inserts + uploads; bypasses RLS) | **`NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`** (anon; browser-safe; must be allowed to **read** rows/URLs per your policies) |
| **Data** | Writes `recordings` rows and `video_url` | Reads the same table and plays `video_url` in the browser |

Set up the Pi pipeline and Supabase Storage + `recordings` table using the capture repo’s README, then point this app at the **same** Supabase project. Ensure **RLS** (or policies) allow **`anon` `select`** on `public.recordings` if the dashboard should list clips publicly, and that **`video_url`** values are playable in a browser (public bucket or signed URLs—see the capture repo’s troubleshooting).

## Stack

- **Next.js** 16 (App Router), **React** 19, **TypeScript**
- **Supabase** (`@supabase/ssr`, `@supabase/supabase-js`) for data and Realtime
- **Supabase Edge Function** `notify-discord` — posts to a [Discord incoming webhook](https://support.discord.com/hc/en-us/articles/228383668-Intro-to-Webhooks) when `recordings` changes (optional; configure in Supabase)
- **Tailwind CSS** 4, **shadcn/ui**-style components, **Lucide** icons

## Prerequisites

- Node.js compatible with Next.js 16 (see [Node.js support](https://nextjs.org/docs/app/getting-started/installation))
- A Supabase project with a `recordings` table (see below)
- `pnpm` (recommended; `npm` / `yarn` work too)

## Environment variables

Create `.env.local` in the project root (do not commit secrets):

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | Supabase **anon** / publishable key (safe for the browser) |

These are read by `lib/supabase/server.ts` and `lib/supabase/client.ts`.

### Edge Function secret (Discord)

The **`notify-discord`** Edge Function reads **`DISCORD_WEBHOOK_URL`** at runtime. That must be set as a **Supabase project secret**, not only in `.env.local`:

- **Dashboard:** Project **Settings → Edge Functions → Secrets** → add `DISCORD_WEBHOOK_URL` (full Discord webhook URL).
- **CLI:** `supabase secrets set DISCORD_WEBHOOK_URL='https://discord.com/api/webhooks/...'` (after `supabase login` and `supabase link`).

Local `.env.local` is for the Next.js app only; the deployed function will return **500** if `DISCORD_WEBHOOK_URL` is missing on Supabase.

## Database

Create a `recordings` table (adjust names if you already use something else—the app expects this shape):

```sql
create table public.recordings (
  id bigint generated by default as identity not null,
  video_url text null,
  created_at timestamp with time zone not null default now(),
  constraint recordings_pkey primary key (id)
);
```

Enable **Row Level Security** and policies that allow your app’s role to `select` (and whatever your ingest pipeline uses for `insert` / `update`). The dashboard only needs read access for listing and playback URLs.

### Live updates (optional)

For **Supabase Realtime** `postgres_changes` on `recordings`, add the table to the Realtime publication, for example:

```sql
alter publication supabase_realtime add table public.recordings;
```

If Realtime is not enabled, the UI still refreshes on an interval and when the window regains focus.

### Discord notifications (optional)

`supabase/functions/notify-discord` accepts **Database Webhook** payloads for `public.recordings` (`INSERT` / `UPDATE` / `DELETE`) and sends a message to Discord. You can also POST a manual test body: `{ "message": "hello" }`.

1. Set **`DISCORD_WEBHOOK_URL`** as an Edge secret (see above).
2. Install the [Supabase CLI](https://github.com/supabase/cli#install-the-cli) (e.g. `brew install supabase/tap/supabase`, or `pnpm add -D supabase` and run `pnpm approve-builds` so the CLI binary installs).
3. Deploy: `supabase functions deploy notify-discord`
4. In Supabase, add a **Database Webhook** on `recordings` that **POST**s to `https://<project-ref>.supabase.co/functions/v1/notify-discord` (use the URL from **Edge Functions** in the dashboard). Ensure the function’s **Verify JWT** setting matches how the webhook calls the URL (often **service role** is sent automatically).

## Usage

1. Point `video_url` at a playable URL (e.g. Supabase Storage public or signed URL). For smooth streaming in the browser, MP4s should be **fast-start** (e.g. `ffmpeg -movflags faststart`) when produced on the Pi.
2. Open the app; recordings appear **newest first**, **5 per page** (`RECORDINGS_PER_PAGE` in `lib/recordings.ts`). Use `?page=` or the on-screen controls to move through pages.

## Project layout (high level)

| Path | Role |
|------|------|
| `app/page.tsx` | Home route: loads a page of recordings, error UI |
| `components/recordings-list.tsx` | List, pagination, Realtime / polling |
| `components/recording-video.tsx` | Video player with loading skeleton |
| `lib/recordings.ts` | Server-side paginated query |
| `lib/supabase/` | Browser and server Supabase clients |
| `supabase/functions/notify-discord/` | Edge Function → Discord webhook |

## Deploy

**Next.js dashboard:** build a production bundle with `pnpm build`, then run `pnpm start` or deploy to any host that supports Node (e.g. Vercel, Railway). Set the same `NEXT_PUBLIC_*` variables in the hosting dashboard.

**Edge Function:** from this repo, with the CLI linked to your project: `supabase functions deploy notify-discord`. Secrets such as `DISCORD_WEBHOOK_URL` are configured in the Supabase dashboard or via `supabase secrets set`, not in Vercel/Next.js env alone.
